# 提案A：信頼性ファースト型
〜安心できる婚活環境の構築〜

## コンセプト
「誰でも入れない」ことを強みにした、信頼性最優先の婚活マッチングWebアプリ。
クレジットカード連携・位置情報・各種書類提出によるマルチレイヤー認証で「安全な出会い」を保証する。

## 解決する課題
- **課題の主体：** マッチングアプリ経験者、特に女性
- **課題の内容：**
  - 「ヤリモクかもしれない」「プロフィールが嘘かもしれない」という不安
  - 会ってみたら全然違った、という経験による疲弊
  - 目的外利用ユーザー（業者・冷やかし）の混入

## ターゲットユーザー
- 25〜35歳、結婚を真剣に考えている男女
- マッチングアプリを1〜3回以上使ったことがある（消耗した経験あり）
- 「もう変な人に会いたくない」と思っている層
- 特に女性側：安心・安全を重視するユーザー

## 主要機能
1. **マルチレイヤー本人確認** — クレジットカード・運転免許証・住所確認を段階的に実施
2. **信頼スコア表示** — 認証レベルに応じてプロフィールに「信頼バッジ」を表示
3. **SNS連携タグ付け** — X/Instagramから趣味嗜好を分析し自動タグ生成
4. **タグベースマッチング** — 共通タグ数や相性スコアを元にマッチング候補を表示
5. **位置情報連携** — 住んでいるエリアのプロフィール記載と実態の整合性確認（任意開示）

## プロダクト形態
Webアプリ（スマートフォンブラウザ最適化）

## ビジネスモデル案
- 男性：月額6,000円
- 女性：月額2,000円
- 追加オプション：認証バッジ強化（年収証明・勤務先証明）は別途有料オプション
※これは仮の方向性です。要検討。

## この提案の特徴
- **良い点：**
  - 「安全」という明確な差別化軸があり、女性ユーザー獲得に有利
  - マッチングアプリ疲れユーザーに強く刺さる
  - クレジットカード連携による自然なヤリモク排除
- **課題・リスク：**
  - 認証の手間がハードルになり、登録完了率が下がる可能性
  - 認証情報の管理に関するプライバシー問題・法的リスク
  - 男性ユーザーの母数が少ないとマッチングが成立しにくい

## 想定されるユーザーフロー
1. メールアドレスで仮登録
2. クレジットカード情報を入力（本人確認 + 課金設定）
3. SNS（X/Instagram）連携 → 趣味タグが自動生成される
4. プロフィール入力・写真登録
5. マッチング候補一覧を閲覧（信頼スコア・共通タグを参考に選択）
6. いいね → マッチング → チャット開始
