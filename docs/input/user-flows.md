# ユーザーフロー

## 1. オンボーディングフロー

### 新規登録
1. トップページにアクセス
2. 「Googleで登録」ボタンをタップ
3. Googleアカウントでの認証（OAuth 2.0）
4. 性別・生年月日・居住エリアを入力
5. **年齢確認（出会い系サイト規制法対応）**
   - 「18歳以上であることの確認が必要です」と案内
   - 公的証明書（運転免許証・パスポート・マイナンバーカード等）の写真を撮影・アップロード
   - 審査完了まで「確認中」ステータスで待機（自動審査 or 手動審査）
   - 審査通過 → 次ステップへ遷移。未成年 → 登録不可として案内
6. 料金プランの確認・決済情報入力（男性：6,000円 / 女性：完全無料）
7. プロフィール作成画面へ遷移

### プロフィール作成
1. 名前（ニックネーム）・プロフィール写真をアップロード
2. 自己紹介文を入力（任意）
3. SNS連携の案内画面
   - 「連携するとマッチング精度が上がります」と表示
   - Instagramの連携ボタン（X連携はPhase 2以降）
4. SNS連携した場合 → AIがSNSデータを解析（バックグラウンド処理）
5. 生成されたタグの確認・修正・追加画面
   - 「音楽好き」「ライブ好き」「カフェ巡り」などのタグが表示される
   - 気に入らないタグは削除、手動で追加も可能
6. プロフィール確認画面 → 「完成させる」で登録完了
7. マッチング候補一覧画面へ遷移

## 2. メインフロー

### マッチング候補閲覧・いいね（男性フロー）
**目的：** 気になる相手を見つけて「いいね（リクエスト）」を送る

**フロー：**
1. ホーム画面でマッチング候補一覧を閲覧
   - 相手のメイン写真・名前・年齢・共通タグ数が表示される
2. カードをタップして詳細プロフィールを確認
   - 「あなたとの共通点：〇個」がハイライト表示
   - 共通タグ一覧・相手のタグ一覧が表示される
3. 「いいね♡」ボタンをタップ → 相手にリクエストとして届く
4. 相手女性がリクエストを承認 → マッチング成立通知が届く
5. 女性からの最初のメッセージを待つ（男性側は送信不可）

**成功条件：** 女性にリクエストが承認され、マッチングが成立すること

---

### いいねリクエスト承認（女性フロー）
**目的：** 届いたいいねリクエストを確認し、気になる相手を承認する

**フロー：**
1. 通知またはリクエスト一覧画面を開く
2. 男性のプロフィール・共通タグを確認
3. 「承認する」または「スキップ」を選択
   - 承認 → マッチング成立通知が双方に届く
   - スキップ → 相手には通知されない（非表示）
4. 承認した相手のチャット画面へ遷移
5. AIアイスブレイカー提案が表示される → 最初のメッセージを送信

**成功条件：** 女性が承認し、最初のメッセージを送信できること

---

### チャット（マッチング後）
**目的：** マッチングした相手と会話を開始・継続する

**女性側フロー（メッセージ送信者）：**
1. マッチング成立通知をタップ → チャット画面へ遷移
2. チャット画面上部に「共通点：音楽好き・ライブ好き」等が表示される
3. AIアイスブレイカーボタンをタップ
   - 「〇〇さんも最近ライブ行ったみたいですね。どのアーティスト好きですか？」などの提案が表示される
4. 提案をそのまま送信 or 編集して送信
5. 以降は双方から自由にチャットを継続

**男性側フロー（返信者）：**
1. 女性からの最初のメッセージ受信通知が届く
2. チャット画面を開き、メッセージに返信する
3. 以降は双方から自由にチャットを継続

**成功条件：** 女性から最初のメッセージが送信され、男性から返信が来ること

## 3. その他のフロー

### ログイン（2回目以降）
1. トップページにアクセス
2. 「Googleでログイン」ボタンをタップ
3. Googleアカウントで認証
4. ホーム画面（マッチング候補一覧）へ遷移

### タグの修正・更新
1. プロフィール編集画面を開く
2. 「趣味タグを編集」をタップ
3. タグの削除・追加・SNS再連携が可能
4. 保存するとマッチングスコアが再計算される

### プロフィール編集
1. マイページ → 「プロフィールを編集」
2. 写真・ニックネーム・自己紹介文・タグを編集
3. 保存して反映

### ブロック・通報
1. 相手のプロフィールまたはチャット画面から「・・・」メニューをタップ
2. 「ブロック」または「通報する」を選択
3. 通報の場合は理由を選択して送信
4. ブロック後は相互に表示されなくなる
