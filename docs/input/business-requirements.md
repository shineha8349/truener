# ビジネス要件

## 1. 課題定義

### 解決する課題
結婚を真剣に考えているマッチングアプリ経験者が、「この人と本当に話が合うのかわからない」という不安を抱えたまま出会い、
マッチング後に会話が続かず自然消滅してしまう問題。

### 課題の深刻度
- マッチングはできても「最初のメッセージが思いつかない」「会話が3往復で終わる」という経験は非常に一般的
- 婚活疲れの主因の一つであり、サービス離脱につながる深刻な問題
- プロフィール欄の「趣味：旅行・映画」といった自己申告情報では、本当の相性が判断できない

### 課題が発生する状況
- マッチング成立直後、最初のメッセージを送ろうとした瞬間
- プロフィールを見て「この人について何を話せばいいかわからない」と感じる場面
- 会ってみたら話が全然続かなかった、という事後的な失望体験

---

## 2. 市場環境

### 既存の代替手段
- Pairs、Omiai、with、タップル、ゼクシィ縁結び等の既存マッチングアプリ
- 結婚相談所（対面サポート型）
- 合コン・友人紹介などのオフライン手段

### 既存手段の課題
- 自己申告プロフィールへの依存：本当の人柄・趣味が伝わらない
- マッチング後のコミュニケーション支援がない：会話のきっかけは全てユーザー任せ
- 「この人は本当に信用できるか」という不安が拭えない

### 競合サービス
| サービス | 特徴 | 課題 |
|----------|------|------|
| Pairs | 国内最大手。安心感訴求が強い | SNS連携なし、会話支援なし |
| with | 性格診断マッチング | SNS由来の趣味分析ではない |
| Hinge | 会話のきっかけ設計が優れている（海外） | 国内普及度が低い |
| Omiai | 趣味タグあり | 手動入力。SNS分析ではない |

## 3. 差別化ポイント

### 競争優位性
- **SNS由来の趣味タグ：** ユーザーが意識していない本音の趣味・嗜好をSNS投稿から自動抽出。自己申告の「盛り」がない
- **共通点の可視化：** プロフィール画面に「あなたとの共通点〇個」を前面表示し、最初のメッセージの壁を下げる
- **AIアイスブレイカー：** 共通タグをもとに最初のメッセージ案をAIが提案。会話継続率の向上に直結
- **女性ファーストメッセージ制（Bumble方式）：** マッチング後は女性からのみ最初のメッセージを送れる。女性が「自分のペースで動ける」安心感を提供し、女性ユーザーの定着率を向上
- **いいねリクエスト承認制：** 男性からのいいねはリクエストとして届き、女性が承認した相手とのみチャットが開始される。不要なメッセージ圧迫がなく、女性の心理的安全性を確保
- **女性完全無料：** 登録・利用の全機能を女性は無料で利用可能。女性ユーザーの母数を最大化し、マッチングの質を高める

### 参入障壁 / Moat
- SNS連携データが蓄積されるほど、マッチング精度が向上する仕組み（データネットワーク効果）
- 一度タグ・履歴が蓄積されると、他サービスへの乗り換えコストが発生（スイッチングコスト）

## 4. タイミング

### Why Now
- SNS（特にX・Instagram）の普及により、ユーザーの趣味・価値観がオープンデータとして存在するようになった
- AI技術の進化（NLP・感情分析）により、SNS投稿からの趣味抽出が現実的なコストで実現可能になった
- マッチングアプリ市場が成熟し、「ただマッチングできる」だけでは差別化できなくなってきた

### 追い風となるトレンド
- 生成AIの活用によるパーソナライズ体験への期待感の高まり
- 婚活・マッチングアプリへの社会的な受容度の向上
- SNSと他サービス連携（OAuth）の一般化

## 6. ビジネスモデル

### 収益モデル
フリーミアム × 月額サブスクリプション（男性課金・女性完全無料モデル）
- 女性は全機能を完全無料で利用可能
- 男性の無料枠：20いいねまで無料
- 男性の課金トリガー：メッセージ送信、および21いいね目以降の利用

### 価格設定
| ユーザー | 月額料金 | 無料枠 | 根拠 |
|----------|----------|--------|------|
| 男性 | 6,000円 | 20いいねまで無料 | 既存マッチングアプリの男性課金相場（3,000〜10,000円）に合わせた設定 |
| 女性 | 完全無料 | 全機能無料 | 女性ユーザーの母数を最大化し、サービスの魅力を高めるため |

### 課金フロー（男性）
1. 登録・プロフィール作成・SNS連携 → 無料
2. いいね送信（〜20回まで） → 無料
3. 21回目以降のいいね送信、またはメッセージ送信 → 月額6,000円の課金が必要
4. 課金後はいいね・メッセージともに無制限

### 主要コスト
- SNS API利用費（X API / Instagram Graph API）
- AI分析エンジン（LLM APIコスト）
- インフラ・サーバー費用
- カスタマーサポート

## 7. 成功指標

### KPI
- 月間アクティブユーザー数（MAU）
- マッチング後のチャット継続率（3往復以上の会話が発生する割合）
- SNS連携率（登録ユーザーのうちSNS連携を行った割合）
- 月次継続率（サブスク継続率）

### マイルストーン
| 時期 | 目標 |
|------|------|
| MVP リリース | Google認証 + SNS連携 + マッチング + チャット の動作確認 |
| 3ヶ月後 | 初期ユーザーからフィードバック収集・タグ精度の改善 |
| 1年後 | クレジットカード認証・年収証明オプションの追加 |

## 8. リスクと仮説

### 検証すべき仮説
1. SNS由来の趣味タグが、ユーザーの「本当の趣味」を正確に反映できる
2. 共通点の可視化により、マッチング後の会話継続率が実際に向上する
3. 女性ユーザーが月額2,000円の課金に納得して登録してくれる
4. SNS連携に対するプライバシー懸念が、登録離脱の大きな原因にならない

### 主要リスク
| リスク | 内容 | 対策 |
|--------|------|------|
| SNS API制限 | X/Instagram APIの仕様変更・費用高騰 | 複数SNS対応・連携なしでも使えるUI設計 |
| プライバシー懸念 | SNSを「見られる」ことへの抵抗感 | 連携内容の透明化・開示範囲の細かい設定 |
| 鶏と卵問題 | 初期ユーザー獲得前はマッチングが成立しにくい | 性別バランスを考慮した招待制・段階的リリース |
| タグ精度 | AI分析が的外れなタグを生成してしまう | ユーザーによるタグ修正機能の提供 |

### 撤退基準
[仮] リリース後6ヶ月でマッチング後チャット継続率が30%を下回り、改善の見込みがない場合
