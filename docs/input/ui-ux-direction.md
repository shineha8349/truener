# UI/UX方針

## 1. 全体の方向性

### デザインテーマ
**フレンドリー・カジュアル**
「出会い」ではなく「つながり」を感じさせる、温かみのある親しみやすいデザイン。
重くなりがちな婚活アプリのイメージを払拭し、SNSアプリのような軽快さを持たせる。

### キーワード
- **温かみ** — 冷たいビジネスライクな雰囲気ではなく、人と人のつながりを感じるデザイン
- **軽快さ** — 操作が気持ちよく、ストレスなくサクサク使える
- **親しみやすさ** — 初めて使う人でも直感的にわかるUI
- **誠実さ** — 婚活というシリアスな文脈を失わない上品さ
- **共通点の喜び** — 「合う人が見つかった！」という発見の楽しさを演出

### 参考サービス
| サービス名 | 参考にする点 |
|------------|--------------|
| Duolingo | フレンドリーなトーン・ゲーミフィケーション的な楽しさ・カラフルだが上品 |
| Slack | チャットUIのシンプルさ・使いやすさ |
| Hinge | マッチング後の会話設計・共通点の見せ方 |
| Instagram | タグ・ビジュアルの見せ方・カジュアルな雰囲気 |

## 2. ビジュアル

### カラー方針
- **メインカラー：** コーラルピンク〜サーモンオレンジ系（#FF6B6B〜#FF8C69）
  - 温かみ・親しみやすさ・恋愛のイメージを表現
- **アクセントカラー：** ソフトパープル（#9B59B6）または明るいターコイズ（#1ABC9C）
  - 共通点・マッチングのハイライトに使用
- **背景：** オフホワイト / 淡いグレー（#F8F8F8）
  - 明るく清潔感のある印象
- **テキスト：** ダークグレー（#333333）
  - 黒すぎず、読みやすい

### タイポグラフィ方針
- **見出し：** 丸みのあるサンセリフ体（Noto Sans JP / M PLUS Rounded 1c）
- **本文：** 読みやすいサンセリフ体（Noto Sans JP）
- **雰囲気：** 硬すぎず、柔らかく、親しみやすい印象

### アイコン・イラスト
- 丸みのある線画アイコン（Heroicons / Lucide React など）
- タグのビジュアルにはかわいい絵文字や小さなイラストアイコンを添える
- イラストは使う場合、シンプルなフラットデザイン

## 3. UX原則

### 重視する体験

1. **「共通点が見つかる喜び」を前面に出す**
   - 候補一覧で「共通点〇個」を常にバッジ表示
   - 詳細プロフィールで共通タグをカラーハイライト
   - マッチング成立時に「〇〇さんと〇個の共通点があります！」というメッセージを表示

2. **「最初の一歩」のハードルを下げる**
   - チャット開始時のAIアイスブレイカー提案は目立つ場所に配置
   - タップ1回で提案が使える・編集できる
   - 送信前プレビューで確認できる

3. **登録フローを短く・快適に**
   - Googleログインでステップを最小化
   - SNS連携は「やると嬉しいこと」として前向きに案内（強制しない）
   - タグ確認画面はスワイプで直感的にOK/NGを選べる

### 避けたいUX

1. **婚活特有の"重さ"・プレッシャー感**
   - 「真剣交際を前提に」「結婚を考えている方のみ」などの重い文言を強調しすぎない
   - フォーム入力が多く、離脱につながる長い登録フローは避ける

2. **SNS連携への不安・不信感**
   - 「何のデータを取得するか」を不透明にしない
   - 連携時に「取得するデータ：投稿テキスト・いいね履歴」を明示する

## 4. 画面構成

### 主要画面一覧
| 画面名 | 目的 | 主要要素 |
|--------|------|----------|
| トップ（未ログイン） | サービス紹介・登録促進 | キャッチコピー・Googleログインボタン |
| 登録フロー① | 基本情報入力 | 性別・生年月日・エリア入力 |
| 登録フロー② | SNS連携 | 連携メリット説明・X/Instagram連携ボタン |
| 登録フロー③ | タグ確認・修正 | 生成タグ一覧・修正・追加 |
| 登録フロー④ | プロフィール作成 | 写真・ニックネーム・自己紹介文 |
| ホーム（候補一覧） | マッチング候補の閲覧 | カード一覧・共通タグ数バッジ |
| プロフィール詳細 | 相手の詳細確認 | 写真・共通点ハイライト・タグ・いいねボタン |
| チャット一覧 | マッチング済みの一覧 | 相手名・最終メッセージ・未読バッジ |
| チャット詳細 | 1対1の会話 | メッセージUI・共通点表示・アイスブレイカー |
| マイページ | プロフィール確認・設定 | 自分のプロフィール・タグ・退会 |

### 画面遷移図

```
[トップ] → [登録①基本情報] → [登録②SNS連携] → [登録③タグ確認] → [登録④プロフィール]
                                                                              ↓
[ホーム（候補一覧）] ←─────────────────────────────────────────────────────────
       ↓
[プロフィール詳細] → [いいね送信]
                          ↓（マッチング成立）
[チャット一覧] → [チャット詳細（アイスブレイカー付き）]

[ホーム] ←→ [チャット一覧] ←→ [マイページ]（ボトムナビゲーション）
```

## 5. インタラクション

### アニメーション方針
- **控えめに使う** — ストレスにならない程度のマイクロアニメーション
- いいねボタンを押したときのハートのアニメーション
- マッチング成立時の紙吹雪または軽いパルスエフェクト
- タグ選択時のチェックアニメーション

### フィードバック
- いいね送信 → ハートアイコンが赤くなり「いいねしました」トースト表示
- マッチング成立 → 全画面エフェクト + 「マッチしました！」モーダル
- メッセージ送信 → 送信済みアイコンが表示される
- SNS連携完了 → 「〇個のタグを生成しました！」と表示

### エラー表示
- フォーム入力エラーは該当フィールドの下にインラインで表示
- ネットワークエラーは画面上部にトースト通知
- SNS連携失敗は「もう一度試す」ボタン付きのモーダルで表示
- 重大なエラー時はエラー専用ページへ遷移（ホームに戻るボタン付き）
