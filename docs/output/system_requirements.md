# システム要件定義書

## システム概要

**Truener**は、X（旧Twitter）・InstagramのSNSデータをAIで解析し、ユーザーの趣味・嗜好タグを自動生成する婚活向けWebアプリである。既存マッチングアプリの課題である「プロフィールだけでは相性がわからない」「マッチング後に会話が続かない」という問題を、SNS由来の共通点可視化とAIアイスブレイカー提案によって解消する。ターゲットは25〜35歳のマッチングアプリ経験者で、女性完全無料・Bumble方式（女性ファーストメッセージ）・いいねリクエスト承認制により女性ユーザーを優先的に獲得し、マッチングの質と量を両立させる。男性は月額6,000円（20いいねまで無料）のフリーミアムモデルで収益を得る。MVPはWebアプリとして提供し、Google認証・SNS連携・タグ生成・マッチング・チャットの5機能に絞ってリリースする。

1. **システムの目的**
   - 解決する課題：自己申告プロフィールへの依存によるマッチング後の会話不成立・自然消滅
   - 目標とする成果：マッチング後3往復以上チャット継続率50%以上・SNS連携率60%以上・月次継続率40%以上
   - 期待されるビジネスインパクト：男性課金ユーザーの安定獲得と、女性無料化による高いマッチング率の実現

2. **主な機能**
   - コア機能：SNSデータからの趣味タグ自動生成・共通点ベースのマッチング・AIアイスブレイカー提案・1対1チャット・Google認証
   - 差別化機能：女性ファーストメッセージ制（Bumble方式）・いいねリクエスト承認制・共通点ハイライト表示
   - 付加価値機能：タグ手動修正・ブロック/通報機能

3. **提供する価値**
   - ユーザーへの直接的価値：「話しかけやすい相手がわかる」体験・最初のメッセージへの心理的ハードルの解消
   - ビジネスへの価値：男性課金による安定収益・女性無料化による差別化と女性ユーザー母数の確保
   - 社会的価値：婚活疲れを軽減し、真剣な出会いをより自然な形で支援する

4. **実現方法の概要**
   - 主要なテクノロジー要素：SNS OAuth連携・LLMを用いたタグ抽出・リアルタイムチャット・Stripeによる課金制御
   - 重要なインテグレーションポイント：X API / Instagram Graph API・Google OAuth 2.0・AI APIサービス・決済サービス
   - スケーラビリティへの考慮：MVP初期は同時接続1,000ユーザー想定。Supabaseのスケールアウト構造とNext.js App RouterのサーバーレスアーキテクチャでPhase 2以降の拡張に対応

## 機能要件

### 1. 認証・ユーザー管理

| 機能 | 詳細 |
|------|------|
| Google OAuth認証 | Google OAuth 2.0によるサインアップ・ログイン。初回ログイン時は登録フローへ遷移 |
| **年齢確認（18歳以上）** | **出会い系サイト規制法に基づき、公的証明書（免許証・パスポート等）の画像アップロードによる年齢確認を登録フローに必須実装。未成年は利用不可** |
| 男女別プロフィール登録 | 性別・生年月日・居住エリア・ニックネーム・写真・自己紹介文の登録 |
| 男女別課金制御 | 女性は全機能無料。男性は20いいね以下かつメッセージ未送信は無料、超過で月額6,000円課金 |
| アカウント管理 | プロフィール編集・退会・データ削除 |

### 2. SNS連携・タグ生成

| 機能 | 詳細 | フェーズ |
|------|------|----------|
| Instagram Graph API連携 | 投稿・いいね情報を読み取り専用で取得 | MVP |
| AIタグ自動生成 | 取得したSNSデータをLLMで解析し趣味タグ（10〜20個）を生成。30秒以内に完了 | MVP |
| タグ確認・修正・追加 | 生成タグのユーザー確認画面。削除・追加・手動入力が可能 | MVP |
| SNS未連携時の手動タグ入力 | SNS連携なしでも手動でタグを設定できるフォールバック機能 | MVP |
| X（旧Twitter）OAuth連携 | 投稿テキスト・いいね履歴・フォロー情報を読み取り専用で取得 | **Phase 2**（APIコストの観点からMVP除外） |

### 3. マッチング

| 機能 | 詳細 |
|------|------|
| マッチング候補一覧 | 共通タグ数・相性スコアで並び替えた候補をカード形式で表示 |
| 共通点ハイライト表示 | 詳細プロフィール画面で「あなたとの共通点〇個」と共通タグをカラーで強調表示 |
| いいね送信（男性→女性） | 気になる相手に「いいね」を送信。男性は20回まで無料 |
| いいねリクエスト承認制（女性） | 女性はリクエスト一覧から「承認」または「スキップ」を選択。承認後にマッチング成立 |
| マッチング成立通知 | 女性が承認した時点で双方に通知 |

### 4. チャット

| 機能 | 詳細 |
|------|------|
| 女性ファーストメッセージ制 | マッチング後、最初のメッセージは女性のみ送信可能。男性は女性からのメッセージ後に返信可能 |
| AIアイスブレイカー提案 | チャット開始時、共通タグをもとにAIが会話の出だしを2〜3パターン提案。そのまま送信or編集が可能 |
| 1対1テキストチャット | マッチング済みの相手とのリアルタイムテキストチャット |
| チャット一覧 | マッチング済み相手の一覧表示（最終メッセージ・未読バッジ付き） |

### 5. 安全・管理

| 機能 | 詳細 |
|------|------|
| ブロック機能 | 任意のユーザーをブロック。双方の画面から非表示 |
| 通報機能 | 不審なユーザーを理由選択付きで通報 |
| SNS連携スコープ最小化 | 取得データを投稿テキスト・いいね履歴に限定し、DM等にはアクセスしない |

## 非機能要件

### パフォーマンス
| 項目 | 目標値 | 根拠 |
|------|--------|------|
| 主要画面レスポンスタイム | 2秒以内 | モバイル利用者が多いため、体感速度を重視 |
| SNS連携・タグ生成 | 30秒以内（バックグラウンド処理） | LLM処理時間を考慮し、非同期処理で対応 |
| チャットメッセージ配信 | 1秒以内 | Supabase Realtimeによるリアルタイム配信 |
| 同時接続ユーザー数 | MVP期：〜1,000人 | クローズドβから段階的に拡大する前提 |

### セキュリティ
- **認証：** Google OAuth 2.0。セッション管理はSupabase Auth / Next.js middlewareで実施
- **SNSトークン管理：** アクセストークンはサーバーサイドで暗号化保存。クライアントには一切渡さない
- **チャット：** メッセージはSupabase Row Level Security（RLS）でマッチングした当事者のみ閲覧可能
- **個人情報：** 個人情報保護法・SNSプラットフォームの利用規約に準拠したプライバシーポリシーを策定
- **SNS連携権限：** 読み取り専用スコープのみ。投稿・フォロー等の書き込みアクセスは要求しない

### スケーラビリティ
- Next.js on Vercelのサーバーレス関数によりリクエスト増加に自動スケール対応
- Supabase（PostgreSQL）はコネクションプールと適切なインデックス設計でPhase 2（数万ユーザー規模）に対応
- AI APIコールはバックグラウンドジョブ化し、タグ生成の負荷をチャット等のリアルタイム処理と分離

### 可用性
- Vercel / Supabaseの標準SLAに準拠（99.9%以上のアップタイムを目安）
- SNS API障害時のフォールバック：手動タグ入力モードに切り替え可能な設計

## 技術スタック

### 推奨スタック

| レイヤー | 技術 | 選定理由 |
|----------|------|----------|
| フロントエンド | **Next.js 14（App Router）** | SSR/SSGの柔軟な使い分け・SEO対応・国内外の採用実績が豊富。モバイルファーストのWebアプリに最適 |
| 言語 | **TypeScript** | 型安全性により実装バグを早期検出。チーム開発・保守性に優れる |
| スタイリング | **Tailwind CSS** | ユーティリティファーストで高速なUI実装。レスポンシブ対応も容易 |
| バックエンド | **Next.js App Router（Route Handlers）** | フロントと同一リポジトリで管理。サーバーサイドでSNSトークンやAI APIを安全に処理 |
| データベース | **Supabase（PostgreSQL）** | RLSによる行レベルセキュリティでチャットデータを保護。Realtimeでチャットのリアルタイム配信も対応。無料枠あり |
| 認証 | **Supabase Auth（Google OAuth）** | Google OAuth統合が容易。Supabaseとのネイティブ連携でセッション管理が簡潔 |
| AI・タグ生成 | **OpenAI API（GPT-4o-mini）** | SNSテキストからの趣味タグ生成・アイスブレイカー提案に活用。コスト効率が高い |
| 決済 | **Stripe** | 月額サブスク・男女別料金制御に対応。Webhookによる課金状態管理が確立されている |
| ホスティング | **Vercel** | Next.jsとの親和性が最高。自動デプロイ・サーバーレス関数・グローバルCDNを標準提供 |
| SNS連携（MVP） | **Instagram Graph API** | MVP時点ではInstagramのみ対応。読み取り専用スコープで最小権限の原則を遵守 |
| SNS連携（Phase 2） | **X API v2** | APIコストの観点からMVP除外。Phase 2でX連携を追加しタグ精度を向上 |
| 年齢確認 | **外部KYCサービス（例：TRUSTDOCK / eKYC各社）またはSupabase Storage + 手動審査** | 公的証明書の画像アップロードと年齢確認。出会い系サイト規制法対応のためMVP必須 |

### アーキテクチャ概要

```
[クライアント（ブラウザ）]
        ↓ HTTPS
[Next.js App Router（Vercel）]
  ├── Server Components（画面描画）
  ├── Route Handlers（API）
  │     ├── SNS OAuth連携エンドポイント
  │     ├── AI タグ生成（OpenAI API呼び出し）
  │     └── Stripe Webhook受信
  └── Middleware（認証・課金状態チェック）
        ↓
[Supabase]
  ├── PostgreSQL（ユーザー・タグ・マッチング・チャットデータ）
  ├── Auth（Googleセッション管理）
  └── Realtime（チャットリアルタイム配信）
```

**目的：** SNSタグマッチングのコアバリューを最小構成で検証する

| 週 | 内容 |

## リスク分析

### 技術的リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| Instagram API の仕様変更・有料化 | 高 | 手動タグ入力のフォールバックを常に維持。Phase 2でX APIを追加し依存を分散 |
| X API コスト（Phase 2以降） | 中 | MVP除外により初期コストを抑制。Phase 2移行時に費用対効果を再評価 |
| AIタグ生成の精度が低い | 高 | ユーザーによるタグ修正機能を必須実装・フィードバックをAI改善に活用 |
| Supabase Realtimeのチャット遅延 | 中 | MVP時点では1秒以内を許容。スケール時にWebSocketへ移行を検討 |
| SNSトークンの漏洩 | 高 | サーバーサイド保存・暗号化・権限スコープ最小化で対応 |

### ビジネス的リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 鶏と卵問題（初期ユーザー不足） | 高 | クローズドβを招待制にして性別バランスを管理。50〜100名規模から開始 |
| 女性ユーザーが集まらない | 高 | 完全無料化・Bumble方式・リクエスト承認制の3施策で女性体験を最優先 |
| 男性が課金に至らない | 中 | 20いいねの無料体験でサービス価値を実感させてから課金に誘導 |

### 法的・コンプライアンスリスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 個人情報保護法への対応不備 | 高 | SNS取得データの利用目的・保存期間・開示手順をプライバシーポリシーに明記 |
| 各SNSプラットフォームの利用規約違反 | 高 | 読み取り専用・ユーザー同意取得・商用利用申請を確実に実施 |
| 出会い系サイト規制法への対応 | 高 | **MVP時点で公的証明書による年齢確認（18歳以上）を必須実装済み**。「婚活・真剣交際目的」の明示もLP・規約に記載 |

### 運用的リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 不審ユーザー・業者の混入 | 中 | ブロック/通報機能・Google認証による最低限の身元確認 |
| AIによる不適切なタグ・メッセージ提案 | 中 | プロンプト設計での制御・ユーザー報告機能・モデレーション体制の整備 |

## UI/UXの方向性

- **デザインコンセプト：** フレンドリー・カジュアル。婚活アプリの"重さ"を払拭し、SNSアプリのような軽快さを実現
- **トーン＆マナー：** 温かみがあり親しみやすい。上品さを保ちながらカジュアルに。プレッシャー感を排除
- **カラーパレット：** メインカラー：コーラルピンク〜サーモンオレンジ（#FF6B6B〜#FF8C69）／背景：オフホワイト（#F8F8F8）／テキスト：ダークグレー（#333333）
- **ターゲットデバイス：** モバイルファースト（スマートフォンブラウザ優先）。PCブラウザでも利用可能なレスポンシブデザイン
- **参考サイト/アプリ：**
  - Duolingo（https://www.duolingo.com）— フレンドリーなUX・マイクロアニメーション
  - Hinge（https://hinge.co）— 共通点を軸にした会話設計・マッチング後のUI
  - Slack（https://slack.com）— チャットUIのシンプルさと使いやすさ
